# Contributing to NEXORA OS

Thank you for your interest in contributing to NEXORA OS.

NEXORA OS is an independent mobile operating system focused on performance, security, privacy, and an intuitive user experience.

## Getting Started

1. Fork the repository.
2. Create a feature branch.
3. Make your changes.
4. Test your changes.
5. Submit a Pull Request.

## Coding Standards

- Write clean and well-documented code.
- Use meaningful variable and function names.
- Keep functions small and focused.
- Document public APIs.

## Commit Messages

Use clear commit messages such as:

```
Add boot animation
Fix memory allocator
Implement touchscreen driver
Improve Aurora UI
```

## Branch Naming

```
feature/<feature-name>
bugfix/<bug-name>
docs/<documentation>
```

## Code Review

All contributions should be reviewed before merging.

## Vision

Every contribution should improve:

- Security
- Stability
- Performance
- User Experience

Thank you for helping build NEXORA OS.
